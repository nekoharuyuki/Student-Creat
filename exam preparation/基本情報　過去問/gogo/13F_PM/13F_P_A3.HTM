問6　

設問　

＜解答＞　ａ　ア　　ｂ　ウ　　ｃ　ア　　ｄ　エ

＜解説＞　関数parse_ml_stringは，タグ付き文字列mlstrを要素に分解して，開始タグの出現順に配列要素elmtblに格納します。この関数parse_ml_stringの中では，関数parse_ml_dataが使われています。関数parse_ml_dataはmlstrへのポインタと整数型変数levelを引数とし，戻り値としてmlstr＋1を返します。この関数では，開始タグ処理，タグの値処理，下位のタグ構造処理，終了タグ処理が行われています。
　開始タグ処理では，要素配列elmtblの配列要素elmtbl［elmnum］．tagにタグ名へのポインタをセットする空欄ａの処理と，配列要素elmtbl［elmnum］．depthに入れ子の深さlevelをセットする処理を行ったあと，'＞'が出現するまでポインタmlstrをインクリメントし，*mlstr='＞'となればタグの値処理に進んでいます。したがって空欄ａでmlstrとして，タグ名へのポインタmlstrをelmtbl［elmnum］．tagにセットすればよい。
　タグの値処理では，要素配列elmtblの配列要素elmtbl［elmnum］．valueにタグの値へのポインタをセットする空欄ｂの処理を行ったあと，'＜'が出現するまでポインタmlstrをインクリメントし，*mlstr='＜'となれば，空欄ｃの処理をして下位のタグ構造処理に進んでいます。elmtbl［elmnum］．valueには，'＞'の次に続くタグの値へのポインタmlstr＋1をセットすればよいので，空欄ｂはmlstr＋1になります。次の開始タグの先頭文字'＜'が出現したとき空欄ｃが行われているので，この空欄ｃで出現した開始タグの個数elmnumを＋1だけカウントアップすればよい。
　下位のタグ構造処理では，while文中でポインタmlstr＋1とlevel＋1を引数として，再帰的に関数parse_ml_dataを繰返して，タグ付き文字列mlstrを要素に分解して，開始タグの出現順に配列要素elmtblに格納します。空欄ｄは下位のタグ構造処理のwhile文の繰返し条件になっています。終了タグの先頭文字'＜'の次の文字'／'が出現するまで繰り返せばよいので，繰返し条件は*(mlstr＋1)！='／'とすればよい。

問8

設問　

＜解答＞　a　イ　　b　カ　　c　ウ　　d　エ

＜解説＞　空欄aを含むメソッドsizeIs( int size)は，サイズの属性値が引数と等しいかどうかを判定するメソッドで，その比較結果をboolean型で戻り値を返す。sizeはint型の基本型変数なので，比較は==で行う。したがって空欄aはthis.size == sizeに該当する。

　空欄bを含むメソッドcolors(String color)は，色の属性値が引数と等しいかどうかを判定するメソッドで，その比較結果をboolean型で戻り値を返す。colorはString型の文字列型変数なので，比較はequals( )で行う。したがて空欄bはthis.color.equals( color)に該当する。

　空欄cはスーパークラスのコンストラクタの呼出しに該当する。
　JeansはスーパークラスPantsを継承して作成されたサブクラスである。クラス定義の冒頭ではコンストラクタを宣言し初期値設定が必要である。解答群にあるsuperについては，

  1. super( ).メソッドの場合は，サブクラス内からスーパークラスを呼出す
  2. super( )単独の場合は，サブクラスからスーパークラスのコンストラクタを呼出す

という機能がある。ここではスーパークラスのコンストラクタの呼出しに使われている。
　コンストラクタJeansの宣言には，4つの引数code, size, color, buttonFrontがあるが，空欄a直後でbuttonFrontについては初期設定があるがそれ以外がない。したがってcode, size, colorについてはスーパークラスのコンストラクタをsuper( code, size, color )に呼出して初期設定する必要がある。したがってウが正解。
　スーパークラスPantsのサブクラスであるJeansにはスーパークラスのメソッドが継承される。メソッドtoString( )は，Pantsでは

public String toString( ) {
return code, + “ , “ + size + “ , “ + color;  
　　}

により定義され，戻り値で返される情報には，code, size, colorの3つしか含まれていない。
　サブクラスJeansでは，buttonFrontに関する情報として文字列”zipper”か”button”を戻り値として返す機能がメソッド　toStoring( )　に必要である。したがってスーパークラスで宣言されたインスタンスメソッド　toString( )　をオーバーライド（再定義）し，サブクラスのメソッドに無い機能だけを追加する必要がある。したがって空欄dは

　super.toString( ) + “ , “ + (buttonFront ? “button : “zipper” )

とすればよい。
