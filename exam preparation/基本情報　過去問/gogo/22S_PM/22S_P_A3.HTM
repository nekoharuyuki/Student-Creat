問9

設問1

〈解答〉　a：エ，b：イ，c：イ，d：エ

〈解説〉

●空欄aについて
空欄aは，英文テキストのファイルから読み込まれた1文字chが空白文字' 'の場合に，出力用行バッファlinebuf[cpos]に改行文字'\0'をセットして改行する条件である。改行文字を含まない，出力する1行の最大文字数は80文字である。配列linbufの先頭の要素番号が0なので，改行文字'\0'は要素番号80の位置に格納することになる。つまりcpos = 80になればlinebuf[cpos]='\0'とすればよいので，空欄aのif文の条件はエのcpos >= COLUMNSが適切。

●空欄b，cについて
改行後，空白文字' 'を行頭に出力するためlinebuf[0]にセットする必要がある。空欄bの直前の処理でcpos = 0にリセットされているので，空欄cでlinebuf[cpos++] = chとして空白文字を行頭にセットしてcposを次の文字の格納位置にインクリメントすればよく，空欄cはイが適切。
最後に出力した空白文字の位置gappを行頭位置0に更新する必要があるが，空欄cのlinebuf[cpos++] = chによりcposをインクリメントする前の空欄bでgapp = cposとして更新すればよく，空欄bはイが適切。

●空欄b，cについて【別解】
英文テキストのファイルから読み込まれた1文字chが空白文字' 'の場合に，cpos >= COLUMNSでない場合は，空欄aに関するif文の処理をスキップし，空欄b，空欄cの処理を行う。この場合，空欄bで最後に格納した空白文字の位置をgapp = cposにセットすればよく，また空欄cでlinebuf[cpos++] = chとして読み込んだ空白文字chを出力用バッファにセットすればよい。

●空欄dについて
単語の途中で1行の最大文字数を超える場合は，その単語が次の行の先頭になるように出力する。英文テキストファイルから読み込んだ1文字が，空白文字でも改行文字でもない場合，cpos >= COLUMNSであれば単語の途中で1行の最大文字数を超えるので，単語の途中で1行の最大文字数を超える単語を除いた出力用行バッファの文字列を出力して改行する必要がある。この処理を行っているのが行番号27～31に該当する。その直後の行番号32の処理で，cposを出力用行バッファlinebufの先頭にリセットしている。
単語の途中で1行の最大文字数を超える単語については，その単語の先頭文字から1行の最大文字数までの部分が，出力用行バッファlinebufの末尾の要素番号i = gapp + 1以降に格納されている。したがってその単語の先頭文字から1行の最大文字数までの部分を，出力用行バッファlinebufの先頭部分に格納し直して出力する必要がある。これを行っているのが行番号33～39の処理に該当する。行番号29の処理で，単語の途中で1行の最大文字数を超えた単語の先頭文字がch1に退避された後，行番号34の処理で単語の先頭文字ch1を出力用行バッファlinebufの先頭に格納し，cposをインクリメントしている。空欄d直後のwhileループで，単語の途中で1行の最大文字数を超える単語について，その単語の2文字目から1行の最大文字数までの部分を，出力用行バッファlinebufの先頭部分に格納し直している。したがってwhileループ直前の空欄dで出力用行バッファの文字位置iを単語の2文字目にインクリメントしておく必要があり，空欄dはエのi++が適切。
