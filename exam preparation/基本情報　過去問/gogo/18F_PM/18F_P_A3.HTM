問6

設問

〈解答〉　a　ウ　　　b　ア　　　c　ウ　　　d　エ　　　e　エ

〈解説〉

空欄aについて

推測数は，4けたがすべて異なる数字を入力する必要がある。空欄aの繰返条件が成り立つ間は，“入力が正しくありません。再度入力してください”のメッセージが出力されているので，空欄aの繰返条件は4けたともすべてが異なる数字ではないという条件になる。したがってウのinValidNumber(num) == FALSEが適切。

空欄bについて

空欄bは推測数numの入力を繰返す条件で，目標数targetと推測数numが完全に一致しない間は繰返し入力する。したがって空欄bはアのisMatch(target，num)が適切。

空欄cについて

空欄cの条件が成り立つ場合に，空欄dの条件が成り立つか否かによりHit数もしくはBlow数をカウントアップしている。推測数の数字num[j]の中に目標数target[i]に一致するものがあれば，このカウントアップを行うので空欄cはウのtarget[i] == num[j]が適切。

空欄dについて

推測数の数字num[j]の中に目標数target[i]に一致するものがあれば，けたが一致していればHit数をカウントアップし，一致していなければBlow数をカウントアップする。空欄dはHit数numHitをカウントアップする条件になっているので，空欄dはけたが一致する条件i == jに該当し，エが適切。

空欄eについて

推測数の4けたとも完全に目標数に一致すれば正解なので，空欄eはHit数numHitけた数DIGITSに一致するという条件になり，エのnumHit == DIGITSが適切。

問7

設問1

〈解答〉　a　イ　　　b　イ　　　c　エ　　　d　カ

〈解説〉

空欄aについて

空欄aを含むPERFORM命令のループ処理で，追加指示のあった氏名について重複番号を1から1ずつ順に増やしながら，氏名キーをもとに電話帳ファイルを読んでいる。このときその重複番号をもつ同じ氏名キーの人が登録されていなければ，その氏名キーのレコードに対して電話番号をセットして電話帳ファイルに書き込めばよい。したがってREAD命令により電話帳ファイルを読んでINVALIDになったときに書き込みを行えばよく，空欄aはイのINVALIDが適切。

空欄b，cについて

空欄b，cを含むPERFORM命令のループ処理で，削除指示のあった氏名について重複番号を1から1ずつ順に増やしながら，氏名キーをもとに電話帳ファイルを読んでいる。このときその重複番号をもつ同じ氏名キーの人が登録されていなければ，削除が不可であり“READ ERROR”のメッセージを表示すればよく，空欄bはイのINVALIDが適切。

重複番号をもつ同じ氏名キーの人が登録されていれば，電話番号が一致する場合は削除を行う。したがってREAD命令により電話帳ファイルを読んでNOT　INVALIDになったときに電話番号が一致するかを判定すればよく，空欄c直後でこの判定を行っているので，空欄cはエのNOT INVALIDが適切。

空欄dについて

削除処理では，電話帳ファイルから氏名と電話番号が一致するレコードを削除した後，同じ氏名の人がほかに登録されている場合は，それ以降の人の重複番号を1ずつ繰り上げる処理が必要である。空欄dを終了条件とするPERFORM命令で重複番号の繰り上げ処理を行っている。重複番号W-BANGOが9を超えれば，このループを終了すればよく，空欄dはカのUNTIL　W-BANGO＞9が適切。

問8

設問

〈解答〉　a　ア　　　b　ウ　　　c　エ　　　d　イ

〈解説〉

空欄aについて

メソッドputは，key又はvalueがnullならば，NullPointerException例外を投げる。空欄aはその条件になっているので，アのkey == null || value == nullが適切。

空欄bについて

メソッドputは，キー（key）と値（value）を1対1に対応付けて登録するメソッドである。keyが既にほかの値と対応付けられていれば，その値をvalueで置き換え，置き換えられる前の値を返す。空欄bを含む処理の前後3行がこの処理に対応する。空欄b直前のremoveメソッドによりreverseオブジェクトからキーに対応する値を削除し，空欄b直後のreturn文により置き換えられる前の値を返している。したがって空欄bを含むputメソッドによりvalueとkeyとの対応をreverseオブジェクトに登録すればよく，空欄bはウのvalue，keyが適切。

空欄cについて

①valueがほかのキーと対応付けられていれば，rermoveメソッドによりreverseオブジェクトからvalueと対応付けられていたkeyとの対応付けを削除する。②removeメソッドの戻り値として，対応付けられていたkeyが返されるので，removeメソッドによりmapオブジェクトからこのkeyに対応するvalueとの対応付けを削除すればよい。したがって空欄cは①に該当する部分であり，エのreverse．remove(value)が適切。

空欄dについて

その他の仕様はメソッドputと同じなので，対応付けを削除した後，メソッドput(key，value)によりkeyとvalueとの対応を登録すればよく，イが適切。

問9

設問1

〈解答〉　a　ウ　　　b　イ　　　c　オ　

〈解説〉

空欄aについて

3行目のLD命令によりC1の転送元ビット列をGR0に格納した後，空欄a直前のSUBL命令によりGR5に16－mを格納し，空欄a直後のSLL命令によりGR0のビット列を(16－m)ビット左論理シフトしている。したがって転送先レジスタGR0の左側mビット以外を0にするには，論理右シフトSRLにより左側mビットのビット列を一旦右端に詰めて，左側mビット以外をオーバーフローにより削除した後，空欄a直後のSLL命令により(16－m)ビット左論理シフトして，元の左端に戻す必要がある。これより空欄aで論理右シフトSRLによりGR0のビット列を(16－m)ビット右シフトし一旦右端に詰めればよく，ウのSRL GR0,0,GR5が適切。

空欄bについて

8行目のLD命令によりC2の転送元ビット列をGR6に格納した後，10行目のSUBL命令によりGR7に16－nを格納し，次の11行目の論理右シフトSRLによりGR6のビット列を(16－n)ビット右シフトし一旦右端に詰めて，左側nビット以外をオーバーフローにより削除している。さらに空欄b直前のSUBL命令によりGR5に(16－m－n)を格納した後，空欄b直後のOR命令によりGR0とGR6との論理和演算を行っている。したがってGR6のnビットのビット列部分を左からmビットの位置にシフトするため，空欄bでGR6のビット列を論理左シフトSLL命令により(16－m－n)ビット左シフトすればよい。これより空欄bはイのSLL GR6,0,GR5が適切。

空欄cについて

3行目のLD命令により転送元のGR0のビット列がGR5に格納されている。空欄c直前のSUBL命令によりGR6に(16－ｍ－n)を格納し，空欄c直後の3行の命令によりGR5のビット列を(16－n)ビット左にシフトして，GR5のnビットのビット列部分を左端に詰めている。したがってGR5の左側(m＋n)ビット以外を0にするため，論理右シフトSRLにより左側(m＋n)ビットのビット列を一旦右端に詰めて，左側(m＋n)ビット以外をオーバーフローにより削除すればよい。これより空欄cはオのSRL GR5,0,GR6が適切。

設問2

〈解答〉　d　ウ　　　e　カ　

〈解説〉

空欄dについて

空欄d直後のOR命令により，GR0にGR7との論理和演算が行われている。GR0には転送元の左側mビット部分以外を0としたビット列が格納されているので，GR7にはGR0の元のビット列の右側(16－m)ビット部分以外を0としたビット列が格納されている必要がある。空欄d直前のSRL命令によりGR7には左側mビットを0とするマスクビットがセットされているので，空欄dでD1の転送先アドレスGR1の元のビット列とGR7との論理積をとりGR7に格納すればよい。したがって空欄dはウのAND GR7,0,GR1が適切。

空欄eについて

GR7には左側nビットを0とするマスクビットが格納されており，空欄e直前のAND命令により転送先アドレスGR3の元のビット列とGR7との論理積の結果，GR7に転送先の元のビット列の右側(16－n)ビット部分以外を0としたビット列が格納される。GR5には左端nビットのビット列部分以外を0とした転送元ビット列が格納されているが，空欄e直後のST命令によりGR5の内容を転送先アドレスGR3に格納しているので，GR5にGR5とGR7との論理和が格納されている必要がある。したがって空欄eはカのOR GR5,GR3が適切。
