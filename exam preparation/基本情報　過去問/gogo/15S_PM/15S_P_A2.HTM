設問

＜解答＞　a　ウ　　　b　エ　　　c　エ　　　d　オ　　　e　ア

＜解説＞接続機器Xと接続機器Yとの距離が230mで，信号伝播速度が230m／マイクロ秒なので，1マイクロ秒後でないと，接続機器Xからのフレームが接続機器Yに到達しない。したがって接続機器Xの送出開始から1マイクロ秒経過しない間は，接続機器Yは接続機器Xからのフレームのキャリアをまだ検知できない。そのため接続機器Xの送出開始から1マイクロ秒経過する前に，接続機器Yから送出するフレームが発生したとき，キャリアを検知できないので，ステップ1からステップ3に移る。したがって空欄aはウ，空欄bはエ。



接続機器Xから送出されたフレームが接続機器Yに1マイクロ秒後に到達する直前に，接続機器Yから送出されたフレームがその1マイクロ秒後に接続機器Xに到達する。したがって接続機器Xはその送出開始後，最大2マイクロ秒後に接続機器Yからの送出フレームを検知する。これより空欄cはエ。
したがってこの2マイクロ秒を経過するより前に，接続機器Xからの送出を終えてしまうと，キャリアの衝突を検知できず，接続機器Xはステップ4からステップ0に移ってしまう。これより空欄dはオ，空欄eはア。

問4　
　
設問
＜解答＞　a　ウ　　　b　オ　　　c　イ　　　d　オ　　　e　エ

＜解説＞逆ポーランド表記法で表された数式は，1次元配列の要素Ex[cp]に1文字ずつ格納されている。繰返し処理の条件Cp≦Lpは，逆ポーランド表記法で表された数式の先頭から1文字ずつ取り出して末尾まで数式を計算する繰返条件である。
配列要素Ex[cp]が数字であれば，T ← T＊10 ＋ ToReal(Ex[cp])の処理により数字1文字を実数型に変換し，位取りを考慮して数値に変換している。
配列要素Ex[cp]が数字以外である場合，以下の(1)から(5)のいずれかにより処理が分岐している。
(1)空欄aの条件，(2)Ex[cp]＝”＋”，(3)Ex[cp]＝”−”，(4)Ex[cp]＝”＊”，(5)Ex[cp]＝”／”
数式は数字以外に，整数の前の空白文字，および”＋” ，”−”，”＊”，”／”から構成されている。空欄aの直後の処理で，負フラグNegF=trueのときにT←−Tをセットし，またNegF=trueでないときの整数TをスタックにPushしている。したがってこれは整数が出現した場合の処理に該当し，空欄aの条件はNumF=trueである。

空欄aの直後の条件NegF＝trueで，負数であるかを判定している。この分岐処理の後のPush命令により，数値Tをスタックにプッシュした直後の処理が，空欄bである。したがって空欄bとして次の数値を取り出すためのTの初期化処理 T ← 0 が必要になる。

空欄cは，配列要素Ex[cp]に”−”が出現した場合，Ex[Cp]が配列の最後尾要素Ex[Lp]でなく，直後の要素Ex[Cp＋1]が数字であるときに行なわれている処理である。つまり負数の手前の符号”−”が出現したときの処理で，NegF ← true に該当する。それに対しEx[Cp]が配列の最後尾要素Ex[Lp]または，直後の要素Ex[Cp＋1]が数字でないときは，演算子の”−”が出現したことになるので，スタックから2つの数値を取り出しY，Xに格納し，Y−Xの値をスタックにプッシュしている。

空欄dは，スタックにプッシュするときの処理に該当する。スタックに積む処理でSpの初期値が0なので，スタックの現在の操作位置Spに対してStack[Sp] ← T とし，スタックに積んだ後，操作位置Spを＋1だけ増やすためSp ← Sp＋1とする必要がある。

空欄eは，スタックからポップするときの処理に該当する。スタックから取り出す場合は，現在の操作位置Spより1つ下に一番上のデータが格納されているので，操作位置Spを−1するためSp ← Sp−1とし，その後でT ← Stack[Sp]として取り出す必要がある。

＜解答＞　イ，ウ，オ

＜解説＞一時ファイル読込み処理ループ中で，発注情報にレコードを追加する処理が行われているが，この発注情報のレコードは(1)拠点コード(2)商品コード(3)発注数(4)配送手段の4つの項目から構成されている。発注数については，一時ファイルAを読込み，このループの処理中で在庫数と受注情報の受注数をもとに設定している。商品コードは，最初の受注情報を記憶する処理で記憶されている。発注情報のレコードは，この商品コード，発注数および一時ファイルAの項目をもとに編集しているので，商品コードと発注数以外の項目である拠点コード，在庫数，配送手段が一時ファイルAに含まれている必要がある。

設問2　
＜解答＞　a　エ　　　b　カ　　　c　イ

＜解説＞輸入拠点の割当ての優先順は以下の順となる。
　　　　　(1)航空便よりも船便で間に合う拠点を優先
　　　　　(2)在庫数が多い拠点を優先
　　　　　(3)拠点コードが小さい拠点を優先
受注数より在庫数が少ない場合は，その拠点の在庫数をその拠点への発注数として，複数の拠点に発注する。
配送日数の余裕は20日とする。
パターン5の場合，拠点コード06の輸入拠点は船便で間に合うが，拠点コード07と08の輸入拠点は船便では間に合わないので，拠点コード06の輸入拠点に船便でまず在庫数70個を発注する。次に拠点コード07の輸入拠点に航空便で70個発注し，残りの60個を拠点コード08の輸入拠点に発注している。在庫数が多い拠点，拠点コードの小さい拠点の順に優先するので，拠点コード08の在庫数aは70より大きくはなく，発注数した60個以上ある必要がある。したがって解答群より在庫数aに該当するのは70個となる。

パターン6の場合，拠点コード09，10，11の輸入拠点はすべて船便では間に合わないので，航空便での発注になるが，在庫数が多い拠点，拠点コードの小さい拠点の順に優先する。したがって拠点コード10の在庫数bは，拠点コード09の在庫数150以下でかつ，拠点コード11の在庫数120以上である必要がある。したがって解答群より在庫数bに該当するのは120個となる。

パターン7の場合，拠点コード13の輸入拠点は船便で間に合うが，拠点コード12と14の輸入拠点は船便では間に合わないので，拠点コード13の輸入拠点に船便でまず在庫数160個を発注する。次に航空便で拠点コード12の輸入拠点に30個発注し，拠点コード14の輸入拠点に10個を発注している。拠点コード14の輸入拠点の在庫数30個に対して10個しか発注していないので，まず拠点コード12の輸入拠点に30個発注し，残り10個を拠点コード14の輸入拠点に発注したと考えられる。拠点コード12の在庫数cが拠点コード14の在庫数30個より少なければ，拠点コード14が優先され，拠点コード14に30個発注されてしまうので，拠点コード12の在庫数cは30個以上である必要がある。したがって解答群より在庫数cに該当するのは30個となる。
