問6

設問

〈解答〉　a：オ，b：エ，c：ウ，d：ア，e：エ，f：ア

〈解説〉

●空欄aについて

空欄aは，障害時に最初に判断すべき項目であり，そのレベルを事前に定義しておき，障害発生時にそのレベルに応じた通報を行う。そのレベルとは障害の影響度のレベルであり，運用責任者が障害時に最初に障害の影響度を判断すべきである。したがって空欄aはオの障害の影響度が適切である。

●空欄bについて

空欄bは，対応状況や回復状況の内容に該当し，解答群からこれに最も近い意味の選択肢を探すと，空欄bはエの障害状況の経緯が適切である。

●空欄cについて

空欄cの項目を基準に問題の解決状況を問題管理委員会で毎週フォローする。表3の改善内容①に，“毎週の問題委員会でその週に解決予定の問題の解決状況を確認する”という記述があり，これを行うには問題の解決予定日時が必要になる。したがって空欄cはウの解決予定日時が適切である。

●空欄dについて

空欄dは，未解決問題で長期間残ってしまい，終了扱いするかどうかの対象となるような問題である。したがってしたがって空欄dは，原因が特定できず，その後再発しない問題で，アが適切である。

●空欄eについて

解決方針が決まった問題は，問題管理プロセスを離れて問題の解決作業として，つまり問題解決のためのシステムの変更作業として変更管理プロセスで実施してしまっている。したがって空欄eはエの変更管理が適切である。

●空欄fについて

未解決の問題は解決日時が空欄となっているので，未解決の問題を重要なものから順に表示するには，解決日時が空欄の問題を障害の影響度の降順に並べて表示すればよい。

設問1

〈解答〉　a：ア，b：ア

〈解説〉

●空欄aについて

資材調達システムの投資回収期間は，開発と運用のための投資を回収できるまでの期間なので，開発による効果が開発費と運用費の合計を上回るまでの期間を計算すればよい。したがって空欄aはアが適切である。

●空欄bについて

資材調達システムの開発によって年間の資材調達費を2％削減できるので，年間の削減効果は2,000百万円×0.02＝40百万円である。開発費が30百万円，年間運用費が2百万円なので，その合計30＋2＝32百万円は1年未満で回収できる。したがって空欄bはアが適切である。

●空欄cについて

営業利益効果＝（4,000－3,000）×0.03－12

●空欄dについて

資材調達費は売上原価の一部を構成しているもので，資材調達費を2％削減すると，売上原価を2,000百万円×0.02＝40百万円削減できる。したがって空欄dはアの売上原価が適切である。

労務費，減価償却費，運用費は販売費に計上するので，労務費5百万円の削減と減価償却費1.6百万円と運用費0.4百万円の発生により，販売費を差引き5－2＝3百万円削減でき，営業利益効果が3百万円になる。したがって空欄eはウが適切である。

●空欄f，gについて

営業支援システムと資材調達システムを開発し，同時に利用した場合，その営業利益効果は以下の式で計算できる。

営業支援システムを開発した場合の営業利益効果は，売上高売上原価率が変わらないという前提で計算したが，資材調達システムと同時に利用した場合，資材調達費を2％削減できるので売上高売上原価率が3,000／4,000＝0.75から（1,000＋2,000×（1－0.02））／4,000＝（3,000－40）／4,000＝0.74に低下する。これにより同時に利用した場合の営業利益効果は，上記の計算のように営業支援システム，資材調達システムを単独に利用した場合の営業利益効果の合計よりも40×0.03＝1.2百万円上回る。 したがって，空欄fはイ，空欄gはイが適切である。

問8

設問

〈解答〉　a：イ，b：イ，c：エ，d：イ，e：エ

〈解説〉

●空欄aについて

図1の行番号7で印字されるx7は，プログラム1の行番号9～14における繰返しループの処理で計算される。i＝6のとき，つまり6回目の繰返し処理における行番号13で計算されるx6－f6／d6→x7の値が，i＝7のとき，つまり7回目の繰返し処理における行番号12のprint命令により印字される。したがってx6－x7＝f6／d6だけの差がある。図1でf6，d6はそれぞれ7.527369×10－9，8.000000なので，x6≠x7である。

●空欄bについて

図2より配列bの要素に格納する値は
　b[n-1] ← n×a[n]
　b[n-2] ← (n-1)×a[n]
　 ：
　b[1] ← 2×a[2]
　b[0] ← 1×a[1]
とすればよいので，空欄bはイのb[k-1] ← k×a[k]が適切。

●空欄c，dについて

行15～18の繰返しループにおいてf，dの値を計算しているが，そのときのxの次数はfよりもdが1だけ小さい。例えばk＝n－2のときの1回目の繰返し処理では，f，dの値を以下のように計算する必要がある。
　f ← (a[n]×x+a[n-1])×x＋a[n-2]
　d ← b[n-1]×x＋b[n-2]
そのためには行15～18の繰返ループに入る直前でf，dの初期値をそれぞれ
　f ← a[n]×x＋a[n-1]
　d ← b[n-1]
としておく必要がある。
したがって，空欄cはエのb[n-1]，空欄dはイのb[k]が適切。

●空欄eについて

（A）の部分の中間結果を印字するプログラムによる同じ処理系での印字結果が－16.000000となり，有効数字7けたでは一致しても，この処理系では，実数型は2進数の浮動小数点形式であって有効けた数は10進数で十数けた程度であるため，実際には演算の過程で丸め誤差が徐々に累積し，（A）の計算結果に16.0を加算するときに，けた落ちが発生したと考えられる。したがって空欄eはエの丸め誤差が適切。

問9

設問1

〈解答〉　a：オ，b：ウ，c：イ，d：エ

〈解説〉

●空欄aについて

空欄aを繰返し条件とするwhileループでは，文字列から多倍長整数を扱う構造体に変換し格納する処理を行っている。すなわち多倍長整数を格納する文字列配列strの文字str[str_idx--]を末尾の下位けたから順に数値に変換し，9けたずつ区切って構造体のメンバの配列要素num->data[num_idx]に格納するごとに，配列要素の要素番号num_idxをインクリメントする処理を繰り返している。
多倍長整数を格納する文字列配列strから9個取り出して配列要素num->data[num_idx]に格納するごとに，whileループ先頭に戻り，配列要素num->data[num_idx]の内容を0に，mulを1にリセットして繰り返す。したがって文字配列strの先頭に達するまではwhileループを繰り返せばよいので，空欄aの繰返条件はオのstr_idx >= 0が適切。

●空欄b，cについて

関数setの最初の処理で，文字配列strの要素番号str_idxの初期値をstrlen(str) - 1とし，文字配列の末尾にセットしている。空欄b，cを含むforループの繰返し処理中で文字配列の要素番号str_idxをデクリメントしている。多倍長整数の文字配列strの末尾から順に数字を取り出し，数値に変換してmul倍し，MP型構造体で表現された多倍長整数numのメンバdata[num_idx]に加算している。したがって，空欄cで位取りのためmulを10倍すればよく，空欄cはイの*= 10が適切。
各配列要素data[num_idx]には，文字配列strの整数を9けたずつ区切って格納するので，空欄bのforループの制御変数iに関する繰返条件はウのi < NUM DIGIT && str_idx >= 0が適切。

●空欄dについて

関数printでは，構造体のメンバdataに格納された多倍長整数を出力する処理を行っている。先ず空欄dのforループ直前で構造体のメンバdataの配列要素の最も大きな要素番号の配列要素に格納された数値をゼロ詰めせずに出力している。したがって9けたずつ区切って構造体のメンバdataの配列要素に格納された多倍長整数を，要素番号の大きい方から順に出力すればよい。つまりforループの繰返し処理で，要素番号num->length - 2から要素番号0までの配列要素の内容を出力すればよい。したがって，forループの空欄dの繰返し条件は，エのi = num->length - 2; i >= 0; i--が適切。

●空欄eについて

●空欄gについて

（2）の場合，1回目の加算321987475＋783917811，2回目の加算500000000＋499999999，4回目の加算734239982＋726325476で繰上りが発生するので，β部分は3回実行され，エが適切。
